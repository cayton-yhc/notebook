# 进制转换

* 首先，判断进制内容————字符？数字？需不需要N--？ *
* 其次，回忆进制转换的要点————一般以十进制为主，我们输出主要是倒序输出余数，所以一般用数组记录余数 *

* 然后写出伪代码——
    准备工作
    循环（当N>0）时
        （N--）
        求余数
        记录每一位余数
        N自除进制

    循环倒序输出 *

* 为什么要自N--？因为有时候进制对应1~基数而非正常的0~基数-1，这时候对N--就可以使得余数也正好对齐，假设进制为26，如26%26==0，但此时0是'a'，要是（26-1）%26就刚好是'z'，而且不会有26/26==1，而是25/26==0*
